Manual de Ingeniería de IA

Conocimiento fundamental para operar una Agencia de IA sin depender de herramientas No-Code.

⚡ Arquitectura de Agentes

Más allá del chatbot simple. Un agente es un LLM con capacidad de **acción**.

  • ReAct Pattern: Reason + Act. El bucle donde la IA "piensa" qué hacer antes de hacerlo.
  • Function Calling (Tool Use): La capacidad nativa (OpenAI/Gemini) de emitir JSON estructurado para ejecutar código real (APIs, DBs).
  • Memoria: Short-term (context window) vs Long-term (Vector DBs).
Stack: LangGraph (Python) o AI SDK (TypeScript).

🔍 RAG (Retrieval Augmented Generation)

Cómo hacer que la IA sepa sobre TU negocio sin alucinar.

  • Embeddings: Convertir texto a vectores numéricos (listas de números) que representan significado.
  • Vector Database: Dónde guardar esos números (Pinecone, ChromaDB, pgvector).
  • Chunking: Estrategia de dividir textos largos. Vital para la calidad de la respuesta.
Tip: El "Hybrid Search" (Keyword + Semántico) es el estándar de oro actual.

🛠️ Freedom Stack: Implementación "Hard Code"

Si n8n desaparece mañana, tu agencia sobrevive así. Desarrollo puro y duro.

1. El Backend (Python/Node.js)

No necesitas una interfaz visual. Necesitas un servidor que escuche eventos.

  • FastAPI / Express: Para crear tus propios endpoints (Webhooks).
  • Celery / BullMQ: Colas de tareas para procesos largos (la IA tarda, no puedes bloquear el servidor).
  • Docker: Empaqueta tu agente en un contenedor y corre igual en tu laptop que en la nube.

2. La Autopista (APIs)

Todo es un POST request.

  • OpenAI API: `POST /v1/chat/completions`. Apréndete los parámetros `temperature` y `top_p`.
  • WhatsApp Business API: Requiere verificar Meta App. Webhooks para recibir, POST para enviar.
  • Stripe/MercadoPago API: Webhooks para activar agentes cuando pagan.

🧠 Estrategia & Unit Economics

💰 Tokenomics

El margen de tu agencia muere si no entiendes esto.

  • Input vs Output Tokens: El input es más barato.
  • Modelo "Cascada": Usa modelos baratos (GPT-4o-mini / Haiku) para triage y caros (GPT-4o / Sonnet) solo para síntesis compleja.
  • Cost per Task: Calcula cuánto te cuesta cada interacción de usuario, no cada mes.

⚖️ Legal & Ética

Protege a tu agencia de demandas.

  • HITL (Human in the Loop): Mecanismo obligatorio para decisiones críticas (ej: reembolsos).
  • Data Privacy: Nunca envíes PII (Información Personal Identificable) al LLM si no tienes un acuerdo BAA (Business Associate Agreement).
  • Alucinaciones: Cláusulas de responsabilidad en tus contratos (como la que pusimos en GenerArise).

La Verdadera Ventaja Competitiva

Cualquiera puede hacer un "wrapper" de GPT.
Tu agencia gana dinero integrando la IA profundamente en los Procesos de Negocio (Business Logic), no solo generando texto.
