GenerArise AI Agency

Guía de Onboarding para Clientes

Requisitos técnicos y de conocimiento para implementar Asistentes de Inteligencia Artificial (Voz y Texto) en cualquier negocio.

Aclaración Importante: La Inteligencia Artificial es tan buena como la información que recibe. Para que el agente responda como un experto, el cliente DEBE proveernos todo el conocimiento interno de su empresa, tarifas, y los accesos a los sistemas donde la IA trabajará.

Fase 1: Base de Conocimiento (Entrenamiento)

1

Data del Negocio (Obligatorio para TODOS los rubros)

El cliente debe entregar un documento (PDF, Word, Notion o sitio web completo) respondiendo:

  • Identidad: Nombre, misión, tono de voz (formal, amigable, persuasivo).
  • Catálogo/Servicios: Lista exacta de servicios, productos, y precios actualizados.
  • FAQ's: Las 20 preguntas más comunes que hacen los clientes y sus respuestas exactas.
  • Políticas: Qué hacer ante devoluciones, cancelaciones, demoras o quejas.
  • Límites del Agente: Qué cosas la IA NUNCA debe prometer o responder.

Fase 2: Conexión de Canales

WhatsApp (Texto y Notas de Voz)

Para conectar el agente a su línea de WhatsApp business o personal:

  • Método QR (No Oficial): Le enviaremos un código QR generado por nuestro sistema (vía Evolution API) que deberán escanear desde su celular en "Dispositivos Vinculados".
  • API Oficial: Deben darnos acceso a su Meta Business Manager, tener una tarjeta de crédito verificada en Meta, y otorgarnos permisos de Administrador de Sistema.

Telegram

Si la empresa usa Telegram para soporte VIP o comunidades:

  • El cliente debe ir a Telegram y escribirle a @BotFather.
  • Usar el comando /newbot y ponerle nombre y usuario al bot.
  • Entregarnos el HTTP API Token que BotFather les genera.

Correo Electrónico

Para agentes que leen y responden emails (Customer Success o Cold Outreach):

  • Google Workspace: Deben ir a seguridad de su cuenta y crear una "App Password" (Contraseña de Aplicación) y darnos ese código + el correo.
  • Herramientas Masivas: Credenciales API de Brevo, Mailgun, o Amazon SES si haremos campañas.

Fase 3: Integraciones por Rubro

Hoteles & Airbnbs

Para que la IA verifique disponibilidad y reserve:

  • PMS API: Token de API de su sistema (Cloudbeds, Mews, Guesty, Hostify).
  • Link a motores de reserva directos.
  • Mapa de habitaciones y check-in/out times.

Real Estate & Inmobiliarias

Para calificar prospectos y mostrar propiedades:

  • CRM Access: API Key de HubSpot, Salesforce, Tokko Broker o EasyBroker.
  • Acceso de editor a Calendly para que la voz AI agende visitas.

Clínicas y Odontología

Para agendar y cancelar turnos médicos por WhatsApp/Voz:

  • Acceso al Software de Turnos Médicos (Webhook o API).
  • Nombres exactos de especialistas y obras sociales aceptadas.

Restaurantes & Delivery

Para tomar pedidos o reservas de mesas:

  • Menú digital exportado en PDF o JSON con precios.
  • Integración con sistema de reservas (OpenTable, CoverManager).
  • Políticas de delivery y mapas de zonas.

Contadores & Finanzas

Para clasificar facturas y recordar impuestos:

  • Carpetas compartidas de Google Drive.
  • Acceso API a sistemas de gestión contable (Xero, QuickBooks).
  • Calendario impositivo anual exacto.

Ticketeras & Eventos

Para soporte de entradas y validaciones:

  • API de la plataforma de Ticketing.
  • Políticas de reembolso estrictas.
  • Listado de eventos actualizados en tiempo real (RSS o JSON endpoint).
✓

Fase Final: Pruebas End-to-End

Una vez que el cliente entrega TODA esta información, nuestro equipo de GenerArise requiere entre 3 y 7 días hábiles para conectar los módulos (Vapi, n8n, OpenAI/Groq), inyectar prompts de base, y realizar pruebas de estrés. No se conectará a la línea oficial del cliente hasta que ellos mismos prueben el bot en nuestro entorno de staging.
